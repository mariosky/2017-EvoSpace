What is an evolutionary algorithm?

It's a Nature-inspired algorithm where a set of data structures representing solutions to a problem evolve via individual or collective changes. Please bear in mind that the words loop, or initialization or many other things are not part of this barebones definition. We do need data structures to represent the solutions and a way to "evolve" them. This is the skeleton. The rest needs to be fleshed out to make this become an actual algorithm that can be run. But this begs the question.

Algorithms + data structures =

Evolution programs

That definition does not throw the algorithm and the data structure in the same bag. There's data somewhere, there's an algorithm somewhere else. They might be the self same thing, but they could be something completely different. If we use the definition from the famous Michalewicz book, you can have many algorithms. One could be an evolutionary algorithm, another could be a completely different one, like PSO. But the problem is where you run them?

Where do we run algorithms?

If you're an straight shooter, the answer would be "A computer". Maybe you could say "the cloud", which is where you run your programs in the XXIst century. But a more precise answer to the question would be in a technosocial system that includes computers and people who own them and which are, at the same time, connected via a social network. Even if you do not consider the human in the loop, where you run the algorithm is going to be a complex system itself, heterogeneous, with different timelines, different capabilities and budgets. If you want to leverage it fully, you will need to take into account this nature.

Population = persistent + ephemeral

Not all the population has to be present all the time. To keep track of the progress of the algorithm, a few select members of the population is quite enough.

Algorithms = any

If there's change and recombination

The zen of genetic algorithms is very general about the mechanisms of change. If there is variability introduced in the population pool, some kind of selection and maybe recombination that is more than enough as long as there is a persistent part of the population updated

Persistent population → Pool

This pool will be an evolving log of the best and the brightest among whatever the rest of the algorithms have found.

Introducing evospace.js

A JavaScript implementation of EvoSpace and a server-side implementation of a pool-based architecture.
Through the introduction of a simple, REST-based interface, Evospace.js can use any kind of resource in the cloud, including volunteers using the browser.
